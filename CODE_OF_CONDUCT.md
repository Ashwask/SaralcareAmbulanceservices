# Code of Conduct

Adapted from the [Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/).

## Our pledge

We — maintainers, contributors, and community members — pledge to make
participation in the Saralcare Ambulance Directory project a respectful,
truth-seeking, and emergency-care-first experience. We hold each other to a
higher standard because the dataset's audience is, by definition, in
distress when they use it.

## Our standards

Examples of behaviour that contributes to a positive environment:

- Treat providers as people running a business, not adversaries. Demand
  evidence for every claim and offer the same courtesy in return.
- Cite sources for every contested change. "I called this number on date X
  and they didn't pick up" is a valid source. "Trust me" is not.
- Disagree with arguments, not with people. Disagree with sourced positions,
  not with motives.
- Translate jargon when speaking to non-technical contributors. The dataset
  serves doctors, drivers, and patients — not only engineers.
- Acknowledge the moderator's authority on first-pass decisions, and use the
  appeal path (`GOVERNANCE.md`) when you disagree.

Examples of unacceptable behaviour:

- Personal attacks, harassment, doxxing, slurs.
- Coordinated bad-faith reporting against a single provider.
- Posting personal contact details (phone, address, ID) of patients,
  drivers, or non-public hospital staff.
- Asserting motive on affiliation edges ("kickback", "exclusive contract",
  "cartel") without a documented public source.
- Promotional posts disguised as contributions. The directory is not a
  marketing surface.
- Inserting commercial intent into a non-commercial workflow (paying for
  badges, paying for ranking, paying for delisting).

## Scope

This Code of Conduct applies to:

- GitHub issues, PRs, discussions, and review comments.
- Email correspondence with the maintainer about the project.
- Public events or talks given on behalf of the project.

It does **not** apply to opinions a contributor expresses in their own
capacity on platforms unrelated to the project. Personal disputes elsewhere
do not become project disputes unless they affect project work.

## Enforcement

Concerns can be raised privately to `ashwinsk@saralcare.com` with subject
`[CONDUCT]`. We will:

1. Acknowledge within 48 hours.
2. Investigate, including (where appropriate) speaking with the people
   involved.
3. Take action proportionate to the situation:
   - **Mild**: clarify expectations, no public action.
   - **Repeated**: temporary restriction on contributions (e.g., PRs require
     pre-review for 30 days).
   - **Severe**: ban from the GitHub organisation, public note in the
     decision log explaining why.

Decisions are logged in `decisions/` with personal identifiers redacted.

## Reporter protection

Reports are confidential. We will not reveal the reporter to the subject of
the report without the reporter's consent. We will not retaliate.

## Honesty

We will admit mistakes publicly. When the maintainer is wrong — about a
record, a decision, a contributor — the correction goes in the public Git
history with reasoning. We expect contributors to do the same.

## Why a Code of Conduct on a directory project?

This isn't a hobby project. People will rely on this data in life-or-death
moments. The standards of accuracy, sourcing, neutrality, and respect we
hold each other to are what makes the dataset trustworthy. The Code of
Conduct is the floor, not the ceiling.
